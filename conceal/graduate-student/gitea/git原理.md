# git 原理

## 内部存储模式

git内部包含了三种数据结构：blob，tree，commit；其中blob用于储存文件内容，tree用于存储文件名字以及文件结构，commit用于存储每一次提交的改变；三者相互配合，就能够实现版本记录的基本功能，当然我们也要知道这只是最基础的部分，git在这之上还做了很多其他的优化。

git内部存储是采用key-value的形式，key是value的hash值（40位16进制）
